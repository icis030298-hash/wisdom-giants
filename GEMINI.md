# GEMINI.md - 안티그래비티 2.0 토큰 절약 및 프롬프트 캐싱 극대화 지침

본 문서는 Google Gemini API의 자동/수동 **Prompt Caching(프롬프트 캐싱)**을 유도하고, 주간 및 분당 API 할당량을 획기적으로 아끼기 위해 안티그래비티 2.0 에이전트가 반드시 준수해야 하는 행동 지침을 정의합니다.

---

## 1. 프롬프트 캐싱(Prompt Caching) 유도를 위한 파일 로드 순서 고정

Gemini의 프롬프트 캐싱은 **프롬프트의 접두사(Prefix)가 정확히 일치할 때** 활성화됩니다. 따라서 매번 파일 읽기 순서가 바뀌거나 프롬프트의 배치 순서가 달라지면 캐시 히트(Cache Hit)에 실패하여 수십만 토큰이 중복 청구됩니다.

에이전트는 대화나 작업 시작 시, 항상 다음 **고정된 순서**로 컨텍스트를 읽고 프롬프트에 조립해야 합니다:

1. **[Prefix 1] 시스템 지침 및 역할 정의** (System Prompt & Core Rules)
2. **[Prefix 2] 프로젝트 맥락 정의서** (`AI_CONTEXT.md` - 프로젝트 전체 개요 및 아키텍처)
3. **[Prefix 3] 에이전트 행동 수칙 가이드** (`GEMINI.md` - 본 문서)
4. **[Prefix 4] 정적 데이터 또는 공통 가이드** (수정되지 않는 마스터 JSON 데이터 등)
5. **[Dynamic] 현재 변경 대상이 되는 단일 타겟 소스코드 파일**

> [!IMPORTANT]
> **순서 고정의 핵심**: 변경될 확률이 가장 낮은 정적 정보(System Prompt -> AI_CONTEXT -> GEMINI.md)를 항상 맨 앞에 배치하고, 실제 수정할 소스코드는 프롬프트의 가장 마지막(Dynamic 영역)에 배치해야 프롬프트 접두사 캐시가 깨지지 않고 유지됩니다.

---

## 2. 컨텍스트 다이어트 (Strict Context Minimalism)

* **타겟 파일 단일 집중**: 무관한 파일(CSS, 레이아웃, 환경 설정 파일 등)을 호기심으로 한꺼번에 읽지 않습니다. 에러 메시지나 작업 범위에 있는 특정 파일 1~2개만 정확하게 타겟팅하여 `view_file`로 조회합니다.
* **와일드카드 검색 금지**: 전체 프로젝트를 재귀적으로 검색하는 것(`list_dir`을 대규모 폴더에 수행하는 행위 등)을 지양합니다.
* **단일 레코드 다이어트**: 99명의 거인 전체 번역을 진행할 때, 99명 전체의 JSON 데이터를 매번 통째로 API 프롬프트에 넣지 마십시오. 오직 작업 대상이 되는 **거인 1명의 세그먼트 데이터**만 슬라이싱하여 프롬프트에 전송합니다.

---

## 3. 에이전트 스파이럴(Subagent) 생성 제한

* **단일 컨텍스트 유지**: 무분별하게 다수의 서브 에이전트를 독립적으로 분기(Parallel Spawning)하면 각 에이전트마다 새로운 캐시 슬롯이 예약되어 토큰 낭비가 극대화됩니다.
* **순차 처리(Sequential Flow)**: 가능한 단일 대화 컨텍스트 또는 동기식 루프 내에서 차례대로 작업을 완수하여 하나의 공통 캐시 컨텍스트를 재사용하도록 유도합니다.

---

## 4. 응답(Response) 크기 압축

* **에코 금지**: API 호출 시 "기존 데이터 전체를 다시 반환하라"고 하지 않고, 오직 "새롭게 생성/수정된 필드만 담은 컴팩트한 JSON을 반환하라"고 요청합니다.
* **비 biography 데이터 생략**: 번역 작업 시 이미 완료된 메타데이터나 기타 다국어 필드는 결과물에서 제외하고, 새로 채워야 하는 필드만 리턴하도록 스크립트를 작성합니다.
